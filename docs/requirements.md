# 機能要件

1. **対象API**
   - 乗算APIと除算APIの2つを提供する。リクエストはHTTP GETとし、クエリストリングで受け取る。
   - 各APIで受け取るパラメータは `A`（被演算子）と `B`（演算子）とし、数値（整数または小数）を想定する。

2. **演算処理**
   - 乗算API：`A × B` を計算し、結果をプレーンテキスト（例: `Result: 6`）で返す。
   - 除算API：`A ÷ B` を計算し、結果をプレーンテキストで返す。B が 0 の場合や非数値入力ではエラー扱いとし、エラーメッセージを返す。

3. **レスポンス仕様**
   - 正常時は HTTP ステータス 200 を返し、本文は計算結果を表すプレーンテキスト。
   - 異常時（0除算、パラメータ欠落、非数値など）も HTTP 200 を返し、本文にエラー内容を任意のプレーンテキストで記載。例: `Error: Division by zero`。
   - クライアントはHTTPステータスでなく本文の文脈で成功/失敗を判定する想定。

4. **入力制御**
   - `A` や `B` に数値以外が入った場合やパラメータが欠落した場合は、それを理由としたエラーメッセージを返却する。
   - 乗算API・除算APIともに、複数パラメータ指定や無意味な文字列は拒否し、エラー本文へ理由を記載する。

5. **利用シナリオ**
   - ブラウザや外部サービスからURLにアクセスするだけで結果画面が表示されることが目標。APIコールは手軽な参照用途を想定し、リクエスト頻度は低い。

6. **補足事項**
   - 出力のプレーンテキストは将来的にフォーマット統一（例: `Result: <数値>` / `Error: <説明>`）する可能性あり。
   - APIドキュメント内でエラー一覧やサンプルリクエスト/レスポンスを別途追記可能。
